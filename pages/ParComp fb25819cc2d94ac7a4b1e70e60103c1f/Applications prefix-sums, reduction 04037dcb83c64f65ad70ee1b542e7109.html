Array compaction: Load balancing

Sequential solution

Input arrays: a , active 

for (i=0; i<n; i++) {
	if (active[i]) {
		a[i] = f(b[i]+c[i]);
	}
}

Work: O ( n + ∣ active ∣ ⋅ f ) \small O(n + |\texttt{active}| \cdot f) O ( n + ∣ active ∣ ⋅ f ) ﻿

where |active| ist the number of indices set to true

Solution: Without array compaction

There are no dependencies between loop iterations → Data parallel computation.

par (0 <= j < p) {
	// i: j*(n/p) ---> (j+1)*(n/p)-1 
	for (i=j*(n/p); i<(j+1)*(n/p); i++) {
		if (active[i]) {
			a[i] = f(b[i]+c[i]);
		}
	}
}

Work: O ( n ) + O ( ∣ active ∣ ⋅ f ) \small O(n) + O( |\texttt{active}| \cdot f) O ( n ) + O ( ∣ active ∣ ⋅ f ) ﻿

Time: O ( ∣ active ∣ ⋅ f ) \small O( |\texttt{active}| \cdot f) O ( ∣ active ∣ ⋅ f ) ﻿

Static assigment of work to processors → load inbalance: processors with active[i] == false do little work.

Solution: With array compaction

Lower work to O ( ∣ active ∣ ⋅ f ) \small O(|\texttt{active}| \cdot f) O ( ∣ active ∣ ⋅ f ) ﻿ by iterating only over active indices:

Split into blocks of size ∣ active ∣ / p \small |\texttt{active}|/p ∣ active ∣/ p ﻿ 

par (i=0; i<n; i++) {
	index[i] = active[i] ? 1 : 0;
}

Exscan(index,n); // exclusive prefix computation

m = index[n-1] + (active[n-1] ? 1 : 0); // m = |active| -> used below

par (i=0; i<n; i++) {
	if (active[i]) {
		compact[index[i]] = i;
	}
}
par (j=0; j<m; j++) {
	i = compact[j];
	a[i] = f(b[i]+c[i]);
}

                0     1      2      3     4      -> n = 5
active  = [true, true, false, false, true]   -> m = 3
index   = [ 1,    1,     0,     0,    1  ]
	↓ Exscan
index   = [ 0,    1,     2,     2,    2  ]

    Now we can copy values with their index for the new compact array in index .

Work: O ( n ) + O ( ∣ active ∣ ⋅ f ) \small O(n) + O(|\texttt{active}| \cdot f) O ( n ) + O ( ∣ active ∣ ⋅ f ) ﻿

Time: O ( n / p ) + Exscan ( p , n ) + O ( ( ∣ active ∣ ⋅ f ) / p ) \small O(n/p) + \text{Exscan}(p,n) + O((|\texttt{active}| \cdot f)/p) O ( n / p ) + Exscan ( p , n ) + O (( ∣ active ∣ ⋅ f ) / p ) ﻿

where Exscan ( p , n ) = O ( n / p + log ⁡ n ) \small \text{Exscan}(p,n) = O(n/p + \log n) Exscan ( p , n ) = O ( n / p + lo g n ) ﻿

Blocking technique

Use when applicable (not always applicable).

Goal: reducing communication / synchronization steps.

  • Blocking technique (general view)
    1. Divide into sub-problems of size n / p \small n/p n / p ﻿ that can be solved sequentially or in parallel.
    1. Assign subproblem to each processor.

      Step 1 and 2 should be fast: ie. O ( 1 ) \small O(1) O ( 1 ) ﻿ , O ( log ⁡ n ) \small O(\log n) O ( lo g n ) ﻿ , ...

    1. Solve subprolems using sequential algorithm.

      Should be perfectly parallelizable: ie. O ( n / p ) \small O(n/p) O ( n / p ) ﻿

    1. Use parallel algorithm to combine subproblem results.

      Should be fast: ie. O ( log ⁡ p ) \small O(\log p) O ( lo g p ) ﻿

      Total cost must be less than O ( n / p ) \small O(n/p) O ( n / p ) ﻿

    1. Apply combined result to subproblem solutions.

      Should be perfectly parallelizable

Blocking technique (alternative view)

  1. Use work-optimal algorithm to shrink algorithm

    Make blocks of size Θ ( n / p ) \small \Theta(n/p) Θ ( n / p ) ﻿ using O ( f ( p , n ) ) \small O(f(p,n)) O ( f ( p , n )) ﻿ time steps per processor.

  1. Use fast and possibly non work-optimal algorithm on shrunk problem

    Solve p \small p p ﻿ subproblems in parallel.

  1. Unshrink, compute final solution with work-optimal algorithm

    Combine into final solution using O ( g ( p , n ) ) \small O(g(p,n)) O ( g ( p , n )) ﻿ time steps per processor.

Usually we can get from O ( n ) \small O(n) O ( n ) ﻿ to O ( n / log ⁡ n ) \small O(n/ \log n) O ( n / lo g n ) ﻿

Resulting parallel algorithm is cost-optimal if f ( p , n ) ∈ O ( n / p ) \small f(p,n)\in O(n/p) f ( p , n ) ∈ O ( n / p ) ﻿ and g ( p , n ) ∈ O ( n / p ) \small g(p,n) \in O(n/p) g ( p , n ) ∈ O ( n / p ) ﻿ .
